当你部署了 Monkeys 应用之后，你可以通过 Prometheus & Grafana 监控 Monkeys 服务的运行状态和性能指标。

## Conductor 指标

> Conductor 是 Monkeys 的流程编排引擎，对外暴露了当前执行工作流数、已完成工作流数、失败工作流数、JVM 内存、数据库连接等指标。

### Prometheus 配置

在 Prometheus 的配置文件中添加如下配置：

- `global.scrape_interval`：监控间隔时间，推荐设置为最短时间间隔（1s），间隔时间设置过长可能会导致无法实时抓取到当前实际正在运行的工作流数。
- `scrape_configs[].metrics_path`：Conductor 服务的 Prometheus 指标接口为 `/actuator/prometheus`。
- `scrape_configs[].static_configs[].targets`：Conductor 服务的地址和端口，你可能需要根据部署的实际情况进行修改。

```yaml
global:
  scrape_interval: 1s

scrape_configs:
  - job_name: 'conductor'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['monkeys-core-conductor:8080']
```

### Grafana 配置

#### 配置 Prometheus 数据源


1. 在 Grafana 控制台的 **连接** - **数据源** 中添加 Prometheus 数据源：

- Prometheus server URL: 填写 Prometheus 服务的地址和端口。
- Scrape interval: 监控间隔时间，推荐设置为最短时间间隔（1s）。
- 点击 **Save & Test** 保存并测试 Prometheus 数据源是否配置成功。

2. 导入监控面板

- 在 Grafana 控制台的 **仪表盘** 页面，点击**新建**按钮。
- 选择**导入**选项。
- 通过仪表板 JSON 模型导入以下 JSON
